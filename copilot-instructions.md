# Create Excuse 要件定義

## 1. アプリ概要

Create Excuse は、ボタンを押すとランダムで「今日の言い訳」を表示するアプリです。

真面目に便利なアプリではなく、少しくだらなくて笑えるアプリにします。

---

## 2. 使用技術

- Next.js
- TypeScript
- Tailwind CSS
- App Router

---

## 3. 画面数

画面は1つだけです。

トップページにすべての機能を実装します。

---

## 4. 画面に表示するもの

トップページには、以下を表示します。

- アプリ名
- 簡単な説明文
- カテゴリ選択
- 言い訳生成ボタン
- 生成された言い訳

---

## 5. カテゴリ

言い訳のカテゴリは以下の4つです。

- 仕事
- 勉強
- 家事
- 運動

---

## 6. 基本機能

### カテゴリ選択

ユーザーは、言い訳のカテゴリを選択できます。

選択肢は以下です。

- 仕事
- 勉強
- 家事
- 運動

### 言い訳生成

ユーザーが「言い訳を作る」ボタンを押すと、選択中のカテゴリに応じた言い訳をランダムで1つ表示します。

### 再生成

もう一度ボタンを押すと、別の言い訳を表示します。

---

## 7. 初期表示

画面を開いた直後は、まだ言い訳を表示しません。

代わりに、以下のような案内文を表示します。

> カテゴリを選んで、今日の言い訳を作ってみましょう。

---

## 8. 言い訳データ

言い訳データはDBに保存しません。

TypeScriptの配列またはオブジェクトで管理します。

例：

```ts
export const excuses = {
	work: [
		"今日はキーボードとの相性が悪いので無理です。",
		"会議で使う集中力を温存しています。",
	],
	study: [
		"教科書がこちらを見てきたので一旦距離を置きます。",
		"脳がまだ起動処理中です。",
	],
	housework: [
		"掃除機も今日は休みたそうな顔をしています。",
		"床との信頼関係を壊したくありません。",
	],
	exercise: [
		"靴ひもが結ばれる覚悟を決めていません。",
		"筋肉とまだ合意形成できていません。",
	],
};
```

---

## 9. 状態管理

外部の状態管理ライブラリは使いません。

Reactの `useState` だけで管理します。

管理する状態は以下です。

- 選択中のカテゴリ
- 現在表示している言い訳

---

## 10. デザイン要件

デザインはシンプルにします。

以下を満たしてください。

- スマホでも見やすい
- ボタンが押しやすい
- 文字が読みやすい
- 余白を適度に取る
- 少しだけゆるい雰囲気にする

---

## 11. 今回やらないこと

以下は実装しません。

- ログイン
- DB保存
- API通信
- お気に入り機能
- SNS投稿機能
- AIによる文章生成
- 複数ページ構成

---

## 12. 完成条件

以下を満たしたら完成です。

- トップページが表示される
- カテゴリを選択できる
- ボタンを押すと言い訳が表示される
- カテゴリに応じた言い訳が表示される
- もう一度ボタンを押すと再生成できる
- スマホ表示でも大きく崩れない
